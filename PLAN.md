# 现金流教育游戏 - V2 迭代开发计划

> 核心目标：从"玩一次学到知识"→"想反复挑战"
> 聚焦两个维度：**内容深度** 和 **游戏节奏感**

---

## 现状摘要（V1 已完成）

- 4个职业、30张事件卡、回合制核心循环、财务报表UI、localStorage存档
- 核心教学完整：资产/负债概念、被动收入、现金流管理
- 主要问题：内容重复（每类仅8张卡）、无时间压力、策略空间有限、无数据反馈

---

## 第一阶段：游戏节奏感重塑

> 让每局游戏有紧迫感、有阶段感、有渐进难度

### 1.1 月数限制与倒计时（game.js / ui.js / index.html）

**问题**：当前可无限玩，无紧迫感
**方案**：
- `game.js` → Game 类新增 `maxMonths = 60`（5年期限）
- `nextMonth()` 中增加超时判定：到期未达成财务自由 → 触发"困在老鼠圈"结局
- `index.html` 头部新增剩余月数倒计时显示 `剩余: XX月`
- `ui.js` → 最后10个月时倒计时变红色闪烁，增加紧迫感
- `css/style.css` → 新增 `.countdown-urgent` 闪烁动画

### 1.2 通货膨胀机制（player.js / game.js）

**问题**：支出永远不变，没有时间压力
**方案**：
- `game.js` → `nextMonth()` 中每12个月（整年结算时）触发通胀事件
- 通胀逻辑：生活类支出（生活费、餐饮、交通）上涨 5~8%，房贷车贷不变
- Player 类的 `expenses` 条目需新增 `inflatable: boolean` 字段区分可通胀/固定支出
- `careers.js` → 每个支出项加上 `inflatable` 标记
- `ui.js` → 通胀时弹出年度总结卡，显示"物价上涨，你的支出增加了"

### 1.3 年度里程碑回顾（game.js / ui.js / index.html）

**问题**：缺少阶段感，玩家不知道自己进步了多少
**方案**：
- `game.js` → 每12个月触发里程碑事件（在通胀之后）
- 新增 `showAnnualReview()` 方法，展示年度数据对比：
  - 今年 vs 去年：现金流变化、被动收入增长、净资产变化
  - 评价等级：A/B/C/D 四档评语
- `player.js` → 新增 `history[]` 数组，每月记录 `{month, cash, cashflow, passiveIncome, totalExpense}`
- `index.html` → 新增年度回顾弹窗 `#modal-annual`
- `css/style.css` → 年度回顾卡片样式

### 1.4 阶段性事件卡解锁（cards.js / game.js）

**问题**：开局就能看到所有投资机会，缺少渐进感
**方案**：
- `cards.js` → 每张 opportunity 卡新增 `unlockMonth` 字段：
  - 基础卡（月份1-12）：指数基金、蓝筹股、自动售货机等小投资
  - 中级卡（月份13-36）：小户型/大户型房产、网店
  - 高级卡（月份37+）：商铺、连锁店、REITs等大型投资（需新增）
- `drawCard()` 函数中按当前月份过滤可用卡池
- 新增高级投资卡 8-10 张（详见第二阶段）

### 2.3 市场波动卡扩充至 12 张（cards.js）

**现有**：6 张
**新增 6 张**：
- 基金分红增加（fund类 incomeMultiplier: 1.15）
- 基金净值下跌（fund类 multiplier: 0.8）

- 经济繁荣（所有资产 multiplier: 1.2，全局性事件）

### 2.4 学习卡扩充至 16 张（cards.js）

**现有**：8 张
**新增 8 张**（进阶概念）：
- 复利效应 / 72法则
- 好负债 vs 坏负债（杠杆教学）
- 分散投资 vs 集中投资
- 通货膨胀对债务人的好处
- ROI（投资回报率）计算
- 应急基金的重要性
- "收入陷阱"——为什么高薪也可能破产
- 税务优化基础

### 2.5 连锁事件系统（cards.js / game.js，新机制）

**问题**：每张卡独立存在，缺少因果关系
**方案**：
- `cards.js` → 新增 `CHAIN_EVENTS` 数据结构
- 当玩家持有特定资产时，可触发后续事件：
  - 持有出租房 → 可能触发"租客拖欠3个月房租"或"房屋装修升级，租金+20%"
  - 持有股票 → 可能触发"公司被收购，股价暴涨200%"或"财报造假，股价归零"
  - 持有生意 → 可能触发"招到好员工，利润+30%"或"被投诉罚款-¥3000"
- `game.js` → `drawCard()` 增加15%概率抽到连锁事件（前提是有对应资产）
- 每个连锁事件带有 `requireAsset` 字段过滤

## 第三阶段：策略自由度提升

> 让玩家有更多操作空间，而非只是被动应对事件卡

### 3.1 主动卖出资产功能（player.js / ui.js / index.html / css/style.css）

**问题**：目前只能在市场波动卡时卖出
**方案**：
- `index.html` → 财务报表的资产列表中，每个资产项右侧增加"卖出"小按钮
- `ui.js` → `updateFinancePanel()` 渲染资产时加上卖出按钮
- 点击后弹窗确认：显示当前市价、关联贷款余额、净收入
- `player.js` → 修复 `sellAsset()` 的字符串匹配问题：
  - 给资产/负债/支出/被动收入统一加 `linkedId` 字段
  - `buyAsset()` 时生成 `linkedId = Date.now() + random`，关联四者
  - `sellAsset()` 通过 `linkedId` 精确匹配，彻底解决脆弱的字符串匹配
- `css/style.css` → 资产项卖出按钮样式

### 3.2 主动还贷功能（player.js / ui.js / index.html）

**问题**：无法提前还清负债，减少月支出
**方案**：
- `index.html` → 负债列表中每项增加"提前还款"按钮
- 逻辑：花费 `liability.total` 一次性还清 → 移除负债 + 对应月供支出
- `player.js` → 新增 `payOffLiability(linkedId)` 方法
- 教学价值：让玩家体会"减少支出 = 加速财务自由"

### 3.3 贷款自然减少机制（player.js / game.js）

**问题**：贷款余额永远不变，不真实
**方案**：
- `player.js` → `processMonth()` 中增加贷款摊还逻辑：
  - 每月月供的一部分用于减少本金（简化为：月供的60%偿还本金）
  - `liability.total -= Math.round(liability.monthly * 0.6)`
  - 当 `total <= 0` 时自动还清，移除负债和对应月供
- 教学价值：让玩家理解分期还款和本金减少的概念

## 第四阶段：数据反馈与成就系统

> 让玩家知道自己"学到了什么"，产生分享欲

### 4.1 游戏结束分析报告（ui.js / game.js / index.html / css/style.css）

**问题**：结束时只有4个数字，没有分析
**方案**：
- `player.js` → 新增决策记录 `decisions[]`，每次买入/卖出/拒绝时记录
  - `{ month, type: 'buy'|'sell'|'reject'|'spend', name, amount, cashflowImpact }`
- `game.js` → 每次决策后调用 `player.recordDecision()`
- `ui.js` → 重构 `showGameOver()`，展示详细报告：
  - 时间线图（文本版）：关键月份的现金流变化
  - 最佳决策 TOP3（按现金流贡献排序）
  - 最差决策 TOP3（按负面影响排序）
  - 投资回报率排名：每个资产的 ROI
  - 学习卡答题统计：正确率、薄弱知识点
  - 综合评语：理财评级 + 个性化建议
- `index.html` → 重新设计 `#modal-gameover` 布局，增加Tab页
- `css/style.css` → 报告卡片、Tab切换、数据展示样式

### 4.2 全局统计系统（storage.js / ui.js / index.html）

**方案**：
- `storage.js` → 新增 `STATS_KEY = 'cashflow_stats'`
- 记录数据：
  - 累计游玩次数、总游玩月数
  - 最快财务自由记录（月数 + 职业）
  - 各职业胜率
  - 累计赚取被动收入总额
  - 学习卡累计答对率
- `index.html` → 主菜单新增"统计数据"按钮和 `#screen-stats` 页面
- `ui.js` → 新增 `renderStatsScreen()` 方法
- 每局游戏结束时自动更新统计

### 4.3 成就系统（新增 js/achievements.js / storage.js / ui.js / index.html / css/style.css）

**方案**：
- 新建 `js/achievements.js`，定义成就列表：

|--------|------|------|------|
| first_asset | 第一桶金 | 首次购买资产 | 一次性 |

| passive_10k | 被动收入破万 | 被动收入≥10000 | 一次性 |
| no_tempt | 定力大师 | 一局中拒绝所有可选消费 | 单局 |
| speed_30 | 闪电自由 | 30月内达成财务自由 | 挑战 |
| speed_20 | 极速自由 | 20月内达成财务自由 | 挑战 |
| all_careers | 全能选手 | 用4个职业各通关1次 | 累计 |
| quiz_master | 财商达人 | 累计答对20题 | 累计 |
| diversified | 分散投资 | 同时持有3种类型资产 | 单局 |
| debt_free | 无债一身轻 | 还清所有负债 | 单局 |

- `game.js` → 关键决策点检查成就解锁条件
- `ui.js` → 成就解锁时右上角弹出 toast 通知
- `index.html` → 主菜单新增"成就"按钮和 `#screen-achievements` 页面

## 第五阶段：技术优化与体验打磨

### 5.1 数据模型健壮性修复（player.js / storage.js）

- **linkedId 关联**（3.1中详述）：替代字符串匹配
- **存档版本控制**：
  - `storage.js` → 存档数据加 `version: 2` 字段
  - 新增 `migrate(saveData)` 方法，v1→v2 自动迁移
  - 加载时检查版本号，不兼容时提示而非崩溃

### 5.2 移动端体验优化（css/style.css / index.html / ui.js）

- 800px以下改为 **Tab切换** 布局（财务报表 / 事件区域），替代当前的上下堆叠
- `index.html` → 游戏界面新增Tab切换按钮 `.mobile-tabs`
- `css/style.css` → 新增 `@media (max-width: 800px)` 下的Tab样式
- 事件卡弹窗在移动端改为底部弹出 (bottom sheet) 而非居中

### 5.3 交互体验细节（ui.js / css/style.css）

- 键盘快捷键：Enter = 下一个月 / Esc = 关闭弹窗
- 数字变化时加过渡动画（现金增减时数字滚动效果）
- 财务自由进度条接近100%时加金色粒子/光晕特效
- 事件卡出现时加翻牌动画

### 5.4 防重复点击保护（game.js / ui.js）

- `ui.js` → `showCard()` 中按钮点击后立即 `btn.disabled = true` 防止重复触发
- `game.js` → `nextMonth()` 入口处同步禁用按钮
