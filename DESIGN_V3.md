# V3 内容升级 · 详细策划设计文档

> 设计原则：不是加功能，而是**用游戏机制让玩家自己体悟出道理**。
> 最好的教学不是告诉答案，而是让玩家在决策中"感受到"差异。

---

## 一、现金流模式可视化（最高优先级）

### 设计目标
让玩家**一眼看出**自己的钱是怎么流动的——是穷人模式、中产模式还是富人模式。这是富爸爸最经典的教学工具，比任何数字都有说服力。

### 核心设计

在财务报表面板顶部增加一个**动态现金流图**，用简化的方块+箭头展示三种模式：

```
穷人模式:     工资 ──────────→ 支出（花光）
              └── 月光族，钱进来就出去

中产模式:     工资 ──→ 负债 ──→ 支出
              └── 赚得多，贷得更多，月供吃掉大部分

富人模式:     工资 ──→ 资产 ──→ 被动收入 ──→ 覆盖支出
              └── 钱生钱的正循环
```

### 判定逻辑（player.js 新增方法）

```
getCashflowPattern() {
    被动收入占比 = 被动收入 / 总收入
    负债支出占比 = 贷款月供总额 / 总支出

    如果 被动收入占比 < 5%  且 负债支出占比 < 20%  → "穷人模式"
    如果 被动收入占比 < 30% 且 负债支出占比 >= 30% → "中产模式"
    如果 被动收入占比 >= 30%                       → "富人模式"
}
```

### UI 设计（HTML + CSS）

在 `.panel-finance` 顶部（在 `<h3>财务报表</h3>` 下方）插入：

```html
<div class="cashflow-diagram">
    <div class="flow-label">你的现金流模式</div>
    <div class="flow-visual" id="cashflow-pattern">
        <!-- 动态渲染 -->
    </div>
    <div class="flow-type" id="flow-type-label">穷人模式</div>
</div>
```

三种模式的视觉表现：

**穷人模式** — 单一直线，灰色调
```
┌─────┐         ┌─────┐
│ 工资 │ ═══════>│ 支出 │
└─────┘         └─────┘
   钱进来就出去，没有任何资产积累
```
- 颜色：暗灰色调，箭头用 `--color-expense` 红色
- 动画：箭头快速流动，暗示钱"一闪而过"

**中产模式** — 经过负债的迂回，黄色警告调
```
┌─────┐    ┌─────┐    ┌─────┐
│ 工资 │ ══>│ 负债 │ ══>│ 支出 │
└─────┘    └─────┘    └─────┘
   赚得越多，贷得越多，月供吞噬收入
```
- 颜色：`--color-liability` 橙色调
- 负债方块大小随负债比例变化（负债越多方块越大）

**富人模式** — 带循环的正向飞轮，绿色+金色
```
┌─────┐    ┌─────┐    ┌──────┐    ┌─────┐
│ 工资 │ ══>│ 资产 │ ══>│被动收入│ ══>│ 支出 │
└─────┘    └─────┘    └──────┘    └─────┘
                ↑          │
                └──────────┘  ← 再投资循环
```
- 颜色：`--color-income` 绿色 + `--color-gold` 金色
- 循环箭头有呼吸动画，暗示"飞轮转起来了"
- 当被动收入 ≥ 支出时，工资方块淡化（暗示：你可以不用工作了）

### 模式切换时的教学提示

当玩家首次从一个模式切换到另一个时，弹出一次性教学 Toast：

- 穷人→中产：「你开始背负贷款了。中产阶级的陷阱是：赚得越多，贷得越多。记住用"好负债"而非"坏负债"。」
- 中产→富人：「恭喜！你的现金流模式开始向富人靠拢了。资产正在为你工作！」
- 富人→中产（退步）：「警告：你的被动收入在下降。检查一下是不是负债增长太快了。」

| ui.js | `updateFinancePanel()` 中调用渲染函数 `renderCashflowDiagram(player)` |
| index.html | `.panel-finance` 内新增 `#cashflow-pattern` 区域 |
| css/style.css | `.cashflow-diagram` 样式、三种模式的配色和动画 |
| game.js | `nextMonth()` 中检测模式变化并触发教学 Toast |

## 二、"先付自己"机制（最高优先级）

### 设计目标
让玩家**亲身体会**投资纪律的力量。不是教他"你应该先付自己"，而是让他在游戏中**选择了不同策略后看到截然不同的结果**。

### 核心机制

每回合开始时（在发薪之后、扣支出之前），弹出一个**投资分配面板**：

```
┌─────────────────────────────────────────┐
│         本月收入分配                      │
│                                          │
│  总收入: ¥12,000                         │
│                                          │
│  ┌──────────────────────────────────┐    │
│  │  ■■■■■■■■■■□□□□□□□□□□           │    │
│  │  先付自己: 30%  (¥3,600)         │    │
│  └──────────────────────────────────┘    │
│                                          │
│  [0%]  [10%]  [20%]  [30%]              │
│                                          │
│  → 投资准备金余额: ¥8,200               │
│  → 剩余可用现金: ¥8,400                 │
│                                          │
│  ⚠️ 先付自己的钱只能用于购买资产          │
│                                          │
│          [ 确认分配 ]                     │
└─────────────────────────────────────────┘
```

### 详细规则

1. **投资准备金（Investment Reserve）**
   - Player 新增字段 `investReserve`（投资准备金余额）
   - 每月选择分配比例（0%/10%/20%/30%），从**总收入**中划入准备金
   - 准备金**只能用于购买资产**（投资机会卡的"买入"按钮检查 cash + investReserve）
   - 准备金**不能用于消费、还贷、支付罚款**（这些只扣 cash）
   - 用准备金买资产时优先消耗准备金，不足部分从现金补

2. **选择0%的后果**
   - 不会弹出分配面板（快速跳过）
   - 但游戏结束时的分析报告会显示："你从未先付自己，如果每月投入10%，你的资产规模将多出 ¥XX"

3. **选择30%的风险**
   - 现金紧张，遇到意外支出（强制支出卡）时更容易破产
   - 但资产积累速度更快
   - 教学点：**先付自己不等于不留应急金，要找到平衡**

4. **首次分配时的教学**（仅第1个月）
   - 弹出富爸爸语录：「富人和穷人的区别不在于收入多少，而在于拿到收入后的第一个动作。穷人先付账单，富人先付自己。」
   - 同时显示：「提示：投资准备金只能用于购买资产，不能消费。这就是"强制储蓄"的力量。」

5. **UI 位置**
   - 不用每月都弹大弹窗（会烦）
   - 第1个月：弹出完整教学弹窗
   - 之后：在游戏头部栏新增一个快捷切换按钮 `[自付: 20%]`，点击可切换比例
   - 头部同时显示准备金余额 `储备: ¥XX`

### 数据支撑——分析报告中的"如果"

游戏结束时计算：
```
假设玩家选的是 X% 先付自己比例
虚拟累计准备金 = 每月收入 × X% × 游戏月数
实际投资总额 = 所有 buyAsset 的 downPayment 之和
差额 = 虚拟累计 - 实际投资总额
如果差额 > 0，显示："如果你坚持先付自己 X%，你本可以多投入 ¥差额 到资产中"
```

## 三、情绪与压力系统（高优先级）

### 设计目标
让"拒绝消费"不再是无成本的理性选择。在现实中，持续压抑消费欲望会导致"报复性消费"。游戏需要模拟这种人性弱点。

### 核心机制：生活满意度

新增一个**生活满意度指标**（0~100），显示在游戏头部。

```
满意度规则:
- 初始值: 70
- 每月自然衰减: -2（生活总是有新欲望）
- 拒绝可选消费: -8
- 接受可选消费: +5（获得满足感）
- 学习卡答对: +3（知识带来自信）
- 买入资产: +2（成就感）
- 达到里程碑（进度每+25%）: +15
- 被动收入首次超过某门槛: +10
```

### 满意度过低的后果

| 满意度范围 | 效果 | 教学意图 |
|-----------|------|---------|
| 60-100 | 正常游戏 | — |
| 40-59 | 事件卡抽到"消费诱惑"概率翻倍 | 压抑会反弹 |
| 20-39 | 触发"报复性消费"事件：强制支付一笔¥2000-5000的消费 | 完全压抑欲望不可持续 |
| 0-19 | "心态崩溃"：本月无法做任何投资决策（跳过投资机会卡） | 极端节俭导致错失机会 |

### 满意度回复方式（让玩家学会"平衡"）

1. **计划性消费**：在满意度低于50时，出现特殊选项"奖励自己"——花¥500-1000的小额消费恢复+15满意度。教学点：**有计划的小奖励好过报复性大消费。**

2. **资产带来的满足感**：每当被动收入达到新的千元整数（1000/2000/3000...），满意度+10。教学点：**真正持久的满足感来自资产增长，而非消费。**

### FOMO（错失恐惧）事件

当玩家拒绝一个投资机会后，有30%概率在2-4个月后触发FOMO事件：

```
「你还记得2个月前放弃的出租小户型吗？
  那套房子现在涨了50%，买了的人每月收租 ¥500。
  你有些后悔——但别冲动！」

  [确实后悔, -5满意度]    [无所谓, 我有自己的计划]
```

选择"无所谓"可获得+3满意度（表示心态成熟）。

教学点：**FOMO是投资者最大的敌人。正确的应对不是后悔过去，而是聚焦当下的策略。**

### "朋友圈"攀比事件（每10个月随机触发一次）

```
「朋友圈刷屏：你的同学刚买了新车/出国旅游/买了大房子。
  你的心里有点不平衡...」

  [跟风消费 ¥3000, +10满意度]    [专注自己的路, -5满意度但+¥0]
```

选"专注自己"后，3个月后会触发后续事件：
```
「你发现那个同学其实贷款买的，每月还款压力巨大。
  而你的被动收入又增长了 ¥XX。谁的路更对？」
  [满意度+8]
```

教学点：**别人的"光鲜"很可能是负债堆出来的。资产和负债的区别，看的不是外表。**

### 涉及文件改动

| 文件 | 改动 |
|------|------|
| player.js | 新增 `satisfaction` 字段 (0-100)、`adjustSatisfaction(delta)` 方法、满意度衰减逻辑 |
| game.js | `nextMonth()` 中每月衰减满意度；拒绝/接受消费时调整；低满意度时修改抽卡权重；FOMO事件调度 |
| cards.js | 新增 FOMO 回调卡和社交攀比卡数据 |
| ui.js | 头部显示满意度指标（笑脸/平脸/哭脸图标+数值）；报复性消费弹窗 |
| index.html | 头部新增满意度显示区域 |
| css/style.css | 满意度指标样式（颜色随数值变化）；低满意度时的UI氛围变化（边框微红等） |

## 四、简化税务系统（高优先级）

### 设计目标
不需要复杂的税法，只需要让玩家**看到数字上的差异**——同样赚1万块，不同渠道到手不同。

### 税率设计

| 收入类型 | 税率 | 对应游戏内收入 | 教学点 |
|---------|------|--------------|--------|
| 劳动收入 | 20% | 工资 | 打工收入税率最高 |
| 资产增值 | 10% | 卖出资产的利润部分 | 买卖差价交税 |
| 被动收入 | 5% | 租金、分红、生意利润 | 持续现金流税率最低 |

### 实现方式

1. **工资税**：每月工资自动扣税。UI中收入栏显示：
   ```
   工资          ¥12,000
   └ 个税(-20%)  -¥2,400
   ```
   实际到手 = 工资 × 0.8

2. **资产增值税**：卖出资产时，利润部分扣10%。
   ```
   卖出价 ¥75,000 - 买入价 ¥50,000 = 利润 ¥25,000
   增值税(10%) = ¥2,500
   实际到手 = ¥72,500
   ```

3. **被动收入税**：每月被动收入仅扣5%。UI中显示：
   ```
   出租房收入     ¥1,500
   └ 税(-5%)     -¥75
   ```

### 教学设计关键——对比

每次发薪时，消息日志自动计算并显示：
```
[第12月] 本月收入总计 ¥13,500
   ├─ 工资 ¥12,000 → 扣税20% → 到手 ¥9,600
   └─ 被动收入 ¥1,500 → 扣税5% → 到手 ¥1,425

   思考：如果同样的 ¥13,500 全是被动收入，到手将是 ¥12,825（多拿 ¥1,800）
```

年度回顾中新增**税务对比栏**：
```
本年累计缴税: ¥XX
├─ 工资税: ¥XX （占 X%）
├─ 被动收入税: ¥XX （占 X%）
└─ 资产增值税: ¥XX （占 X%）

💡 富爸爸说："税法是奖励投资者的。你缴的劳动税越多，说明你还在用时间换钱。"
```

| 文件 | 改动 |
|------|------|
| player.js | 新增 `TAX_RATES` 常量；修改 `processMonth()` 中收入结算加入税后计算；`sellAsset()` 中利润部分扣增值税；新增 `taxPaid` 跟踪对象 `{salary, passive, capital}` |
| game.js | `nextMonth()` 日志显示税后明细 |
| ui.js | 收入列表渲染时显示税率和扣除额；年度回顾增加税务分析 |
| css/style.css | 税额显示样式（灰色小字，缩进） |

## 五、象限进化 E→S→B→I（高优先级）

### 设计目标
让玩家体验从打工者到投资者的**身份转变**，理解财务自由的真正路径不只是"买资产"，而是"改变你的象限"。

### 四象限定义

| 象限 | 身份 | 收入来源 | 解锁条件 | 特点 |
|------|------|---------|---------|------|
| E（雇员） | 打工人 | 工资 | 初始状态 | 稳定但有天花板 |
| S（自雇） | 个体户 | 自雇收入 | 现金≥50000且有≥1个生意类资产 | 收入更高但不稳定 |
| B（企业主） | 老板 | 系统收入 | 有≥3个生意类资产且被动收入≥5000 | 生意为你工作 |
| I（投资者） | 投资家 | 投资收入 | 被动收入≥总支出的80% | 钱为你工作 |

### 进化机制

到达条件后，下一回合开始时弹出**象限进化选择**：

```
┌──────────────────────────────────────────────────┐
│               象限进化机会！                        │
│                                                    │
│  你已经具备了进入 S象限（自雇者）的条件。            │
│                                                    │
│  E → [S] → B → I                                  │
│                                                    │
│  进化效果:                                         │
│  ✓ 工资消失，替换为"自雇收入"（波动范围: ±30%）      │
│  ✓ 解锁独有投资机会卡（成本更低、回报更高）          │
│  ✓ 月收入上限提高                                   │
│  ✗ 收入不再固定，每月在基础值的70%-130%波动           │
│  ✗ 需要承担经营风险                                 │
│                                                    │
│  [接受进化]        [留在 E 象限]                     │
└──────────────────────────────────────────────────┘
```

### 各象限特殊机制

**E象限（雇员）** — 默认状态
- 固定月薪，零风险
- 投资机会较少，金额较小
- 教学：安全但没有杠杆

**S象限（自雇者）** — 可选进化
- 无固定工资，改为"自雇收入"，每月在 `baseSalary × 0.7 ~ 1.3` 之间随机
- 解锁S象限专属卡：个人工作室、接外包项目、技能培训等
- 某些月份可能收入为0（没接到单）——教学：自雇不等于自由
- 教学点：**S象限的人是自己给自己打工，还是在用时间换钱，只是老板换成了自己**

**B象限（企业主）** — 条件解锁
- 自雇收入稳定化（系统化运营，波动缩小到 ±10%）
- 生意类资产收入+20%加成（你知道怎么管了）
- 解锁B象限专属卡：招聘团队、开分店、品牌授权
- 教学点：**B象限的人建立系统，让系统为你赚钱。你可以休假，但钱不会停。**

**I象限（投资者）** — 最终形态
- 所有被动收入+10%加成（投资眼光更准）
- 解锁I象限专属大型交易卡：天使投资、企业并购
- 此时如果被动收入 ≥ 支出，即达成财务自由
- 教学点：**I象限的人用钱生钱，这是财务自由的终极形态。**

### 象限显示

游戏头部新增象限指示器：
```
[E] → S → B → I      第 15 月 | 剩余 45月
```
当前象限高亮金色，后续象限灰色，已经历的象限绿色打勾。

## 六、复利可视化与后悔计算器（中等优先级）

### 设计目标
让时间的价值变得**可感知**。小白最大的错误是"等我有钱了再投资"，这个设计要让他们看到早投资一个月和晚投资一个月的差距。

### 机制一：资产累计收益追踪

每个资产卡片上显示"累计已为你赚了多少钱"：

```
资产列表:
├─ 出租小户型  ¥50,000  [卖]
│  └ 累计收入: ¥6,000 (持有12个月)  ROI: 60%
├─ 蓝筹股     ¥20,000  [卖]
│  └ 累计收入: ¥800 (持有8个月)  ROI: 4%
```

需要在 asset 上增加 `purchaseMonth` 和 `totalEarned` 字段。每月 `processMonth()` 时累加。

### 机制二：错过的机会计算器（年度回顾中）

年度回顾新增一个"如果"板块：

```
本年你拒绝了 3 个投资机会。如果你全部买入:
├─ 指数基金 (第3月放弃): 至今已可赚 ¥540
├─ 小户型出租房 (第7月放弃): 至今已可赚 ¥500
└─ 潜在累计多出收入: ¥1,040

💡 时间是资产最好的朋友。越早投资，复利越有力量。
```

需要在 `decisions` 数组的 `reject` 类型记录中保存被拒卡片的 `monthlyIncome`，在年度回顾时计算 `(当前月 - 拒绝月) × monthlyIncome`。

### 机制三：72法则实时提示

当玩家在投资机会卡上悬停或查看详情时，自动计算：

```
这项投资的年化回报率: 约 7.2%
按72法则: 72 ÷ 7.2 ≈ 10年翻倍
```

## 八、协同效应系统（中等优先级）

### 设计目标
教会玩家"1+1>2"的道理——专注一个领域深耕，资产之间会产生协同效应。

### 协同组合表

| 组合 | 需要的资产 | 加成效果 | 教学点 |
|------|-----------|---------|--------|
| 房产集群 | ≥3个realestate类资产 | 所有房产收入+15%（批量管理降成本） | 规模效应 |
| 生意帝国 | ≥3个business类资产 | 所有生意收入+20%（品牌协同） | 品牌效应 |
| 投资组合 | 同时持有stock+fund+realestate | 所有被动收入+10%（风险分散奖励） | 分散投资 |
| 仓储+网店 | 同时持有仓储中心+网店 | 两者收入各+25%（供应链协同） | 产业链整合 |
| 全产业链 | 同时持有≥5个不同资产 | 所有收入+8% | 多元化经营 |

### 触发方式

每次 `buyAsset()` 后检查是否触发了新的协同组合。首次触发时弹出Toast+特殊音效：

```
🔗 协同效应触发！
"房产集群" — 你拥有3个以上房产，批量管理降低了成本。
所有房产收入 +15%！
```

### 实现方式

协同效果**不修改原始资产数据**，而是在 `getPassiveIncome()` 中计算时叠加：

```javascript
getPassiveIncome() {
    let base = passiveIncomes.reduce(sum);
    let bonus = this.calculateSynergyBonus();
    return base + bonus;
}
```

这样卖出资产后协同自动失效，不需要额外清理。

### 涉及文件改动

| 文件 | 改动 |
|------|------|
| player.js | 新增 `SYNERGIES` 常量、`calculateSynergyBonus()` 方法、修改 `getPassiveIncome()` |
| game.js | `handleOpportunity()` 买入后检查协同触发 |
| ui.js | 财务报表收入区增加"协同加成 +¥XX"行；Toast通知 |
| css/style.css | 协同加成行的特殊样式（金色闪光） |

## 九、资产保护机制（低优先级）

### 设计目标
教会玩家"进攻赚钱、防御守钱"。这是多数财商入门内容忽略的点。

### 核心机制：资产保护等级

```
等级0（无保护）: 默认状态，所有风险事件全额承受
等级1（基础保护 ¥3,000）: 购买综合保险，意外支出减半（已实现）
等级2（中级保护 ¥8,000）: 成立个人公司持有资产，诉讼/罚款类事件伤害-50%
等级3（高级保护 ¥15,000）: 家族信托结构，资产价值波动影响-30%
```

### 新增风险事件卡（触发保护机制）

| 事件 | 无保护损失 | 等级2损失 | 等级3损失 |
|------|-----------|----------|----------|
| 合伙人纠纷诉讼 | 失去1个资产 | 赔偿¥5000 | 赔偿¥2500 |
| 经济危机 | 所有资产-30% | 所有资产-30% | 所有资产-20% |
| 政策变动（某类资产限制） | 被迫出售 | 被迫出售但免税 | 可选择保留 |

### 保护等级购买时机

游戏进行到30月后，出现"财务顾问"事件卡，推荐购买保护。这个时间点玩家通常已经有一定资产，需要开始考虑防御。

教学点：**赚到第一桶金后，保护它和继续增长一样重要。很多人赚到了钱但因为缺乏保护最终失去了一切。**

### 涉及文件改动

| 文件 | 改动 |
|------|------|
| player.js | 新增 `protectionLevel` 字段 (0-3) |
| cards.js | 新增3张保护购买卡 + 3张风险事件卡 |
| game.js | 风险事件处理时根据保护等级减免 |

## 十、破产重启机制（低优先级）

### 设计目标
传达"失败是学费"的理念。破产不是终点，而是带着经验重新开始。

### 核心机制

破产时，替代直接结束，给玩家选择：

```
┌─────────────────────────────────────────┐
│        你破产了...但还没结束              │
│                                          │
│  富爸爸说："我破产过两次，但我从未                │
│  变穷过。破产是暂时的，贫穷是永远的。"    │
│                                          │
│  你可以选择:                              │
│                                          │
│  [东山再起]                               │
│  • 资产清零，但保留所有学到的知识          │
│  • 学习卡已答对的题不再重复               │
│  • 财商等级保留                           │
│  • 可以选择新职业                         │
│  • 剩余月数不变（时间继续倒计时）          │
│                                          │
│  [接受失败，回到主菜单]                    │
│  • 查看本局分析报告                       │
│  • 记录到统计数据                         │
│                                          │
└─────────────────────────────────────────┘
```

### 重启规则

- 清空：资产、负债、被动收入、投资准备金
- 保留：`financialIQ`（财商等级）、`quizCorrect/quizTotal`（答题记录）、`decisions`（用于最终报告对比）
- 重置：现金重置为新选职业的初始现金、月份不变
- 特殊：已答对的学习卡ID存入 `answeredQuizIds[]`，不再抽到重复题

### 限制

一局只能重启**一次**。第二次破产直接结束。教学点：**失败后的机会不是无限的，每一次重来都更珍贵。**

### 涉及文件改动

| 文件 | 改动 |
|------|------|
| player.js | 新增 `restart(newCareer)` 方法（清空资产保留知识）、`restartCount` 字段、`answeredQuizIds[]` |
| game.js | `endGame(false)` 中判断是否可以重启，弹出选择 |
| cards.js | `drawCard()` 的learning类型过滤已答对的题目 |
| ui.js | 破产重启选择弹窗 |
